Purpose

The purpose of this document is to define the requirements for the development of FRH tool that will handle CRUD operation on trainee information.

Target Audience

The intended audience for this Tool can be anyone who wants to manage a team, keeping a record of the information of all the members of a team or group.

User Interface Elements

There will be three views:

  1. Main View: Contains all the trainee information in a grid and three buttons Add, Edit, Delete menu for the action.
  2. Second View: Same for Add and Update operation which contains all the information of a trainee and add, edit, clear buttons.
  3. Third View: Contains all the information in read-only format, and four buttons (first, last, previous, next).

Additional user interface guidelines are provided for enabling/disabling buttons based on user interactions.

Input/Output Requirements

In input, the data of any person will be entered in various text-boxes and then will be stored in a flat file. The Add button appends information to the file, and the Update button modifies existing data in the file.

Functional Requirements

Detailed functional requirements include specifications for Main View, Second View, and Third View, along with input validation, file operations, and user interaction guidelines.

Non-functional Requirements

Non-functional requirements specify constraints such as file locking, file format, and additional requirements for the FHP Project.

Additional Requirements

  1. Filter Condition
  2. DateTimePicker
  3. Authentication Form

These additional requirements extend the functionality of the FHP Project to include advanced search, improved DateTimePicker, and user authentication features.
